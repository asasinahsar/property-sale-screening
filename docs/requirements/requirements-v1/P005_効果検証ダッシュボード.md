# 要件定義書: P005 効果検証ダッシュボード

## 目的

business-requirements の成功基準（工数削減・品質標準化・抜け漏れ防止・トレーサビリティ）を計測・可視化する。山田ペルソナ（責任者）の「効果検証」ステップの受け皿。ハイブリッド方式：導出系（カバレッジ・トレース可能率・再現性・平均構造スコア）はシステム自動算出、工数削減はタイムログ手入力で補完する。

> **スコア呼称**: 構造スコア（定量）／イベントスコア（定性）／総合スコア（派生値）。
> **実行権限**: 閲覧は担当者・責任者とも可（主利用は責任者）。工数ログ入力は担当者・責任者とも可。
> **粒度の注記**: 本フェーズは概念レベルのIPO＋UI状態遷移。テーブル/API/集計式は次のDB/API設計フェーズで詳細化。導出元は [`scoring-and-pipeline.md`](./scoring-and-pipeline.md) 及び既存データ（母集団フラグ・出典・実行履歴）。

## 機能

| # | 機能 | 説明 | Phase |
|---|------|------|-------|
| 1 | 効果検証KPI表示 | 工数削減率・品質標準化（再現性）・母集団カバレッジ・トレース可能率・平均構造スコアをカード表示 | Phase1 |
| 2 | 工数ログ入力 | タスク種別・所要時間（分）を手入力し工数削減の計測元を蓄積 | Phase1 |
| 3 | 期間フィルタ・トレンド表示 | 対象期間で絞り込み、KPIの期間推移を表示 | Phase1 |

## Input / Process / Output

| 機能 | Input | Process | Output |
|------|-------|---------|--------|
| 効果検証KPI表示 | 母集団フラグ・スコア結果・出典・実行履歴・工数ログ | カバレッジ/トレース可能率/平均構造スコア/再現性を導出、工数削減率を集計しスナップショット化 | KPIサマリ（工数削減・品質標準化・カバレッジ・トレース可能率・平均構造スコア） |
| 工数ログ入力 | タスク種別・所要時間（分）・対象期間/実行 | 検証後に工数を記録 | 更新済み工数ログ |
| 期間フィルタ・トレンド表示 | 対象期間 | 期間でKPIスナップショットを抽出・推移を整形 | KPIトレンド（期間推移） |

## UI状態遷移

| 要素 | 対応機能 | 状態 |

| KPIカード群 | 効果検証KPI表示 | ローディング／表示／データなし |
| 工数ログ入力フォーム | 工数ログ入力 | 通常／入力中／保存中／保存済み／検証エラー |
| 期間フィルタ | 期間フィルタ・トレンド表示 | 未適用／適用中 |
