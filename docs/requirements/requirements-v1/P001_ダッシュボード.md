# 要件定義書: P001 ダッシュボード（兼 企業ランキング一覧）

## 目的

ログイン後の入口画面。KPIカードと直近イベントで全体を把握しつつ、構造スコア（定量）×イベントスコア（定性）を統合した企業ランキングを閲覧・絞り込みし、深掘り対象（ロングリスト）の選定を開始する。スクリーニングの実行／再読込もここから行う。評価・ランキングの単位は「企業」。

> **スコア呼称**: 構造スコア（=定量）／イベントスコア（=定性）／総合スコア（両者を統合した派生値・ランキングの並び順キー）。
> **粒度の注記**: 本フェーズは概念レベルのIPO＋UI状態遷移。具体的なテーブル名・API仕様・スコア計算式の係数は次のDB/API設計フェーズで追記する。Inputの「構造スコア・イベントスコア・各指標・定性シグナル」の生成元は [`scoring-and-pipeline.md`](./scoring-and-pipeline.md) を参照し、本画面は結果を受け取る前提。
> **実行権限**: 閲覧・スクリーニング実行・ロングリスト追加は **担当者・責任者ともに可**（ロール制限なし）。
> **異常系の共通方針**: 処理失敗時は「エラー表示＋再試行導線」を共通パターンとする（通知連携はPhase2）。
> **バリデーション**: 自然言語クエリの最大長は200文字。

## 機能

| # | 機能 | 説明 | Phase | 仕様書からの展開／変更点 |
|---|------|------|-------|--------------|
| 1 | KPIカード表示 | 対象企業数・高構造スコア社数・平均構造スコア・直近イベント件数を表示 | Phase1 | 「高スコア社数」→「高構造スコア社数（しきい値=構造スコア70以上）」に改定 |
| 2 | 直近イベントバナー | 直近7日の新規イベント企業／イベントスコア（Phase1スナップショット） | Phase1 | Phase1=静的データ参照を明記 |
| 3 | スクリーニング実行／再読込 | 一括スコアリングを起動しランキング・KPIを更新 | Phase1 | 非同期処理・進捗状態 |
| 4 | 企業ランキング表示 | 総合スコア順に企業を一覧（列は下記で確定） | Phase1 | 表示列を確定 |
| 5 | フィルタ・並び替え | 業種・時価総額帯・スコア帯・イベント有無で絞り込み。並び替えのデフォルトは総合スコア順、構造スコア順／イベントスコア順へトグル可 | Phase1 | 並び替えトグルを確定 |
| 6 | 企業検索（完全一致＋自然言語） | 企業名/証券コードの完全一致検索に加え、自然言語クエリをAIエージェントが解釈する自然言語検索モード | Phase1 | 自然言語検索モードを追加 |
| 7 | 詳細へ遷移 | 行選択でP002へ | Phase1 | 変更なし |
| 8 | ロングリストへ追加 | チェックした企業をP003へ一括追加 | Phase1 | 一括追加（複数選択） |
| 9 | 確信度インジケータ | データ充足度が低い企業をスコアセルに控えめなバッジ/トーンで明示（列は増やさずインライン表現） | Phase1 | 新規。確信度はscoring-and-pipeline.md側で算出 |
| 10 | イベントブースト倍率バッジ | イベントによる加点倍率（例「×1.30」）を企業名横に表示 | Phase1 | 新規。倍率の算出はscoring-and-pipeline.md側 |
| 11 | スコア変化の差分ハイライト | 前回スクリーニング比でスコア上昇/下降を矢印・色で強調 | Phase2 | 新規（履歴データ前提） |

### ランキングテーブル表示列（確定・Phase1）

| 列 | 内容 |
|----|------|
| 企業名 | 行ラベル（左端）。横にイベントブースト倍率バッジ・低確信度マークをインライン表示 |
| 業種 | 業種区分 |
| 構造スコア | 定量指標を合成したスコア（0–100） |
| イベントスコア | 定性シグナルを合成したスコア |
| PBR | 株価純資産倍率 |
| 含み益（億） | 賃貸等不動産の含み益（億円） |
| イベント | 直近イベント有無/概要 |
| 詳細ボタン | P002へ遷移 |

## Input / Process / Output

| 機能 | Input | Process | Output |
|------|-------|---------|--------|
| KPIカード表示 | 最新スコアリング結果（scoring-and-pipeline.md） | 対象企業数・高構造スコア社数（構造スコア70以上）・平均構造スコア・直近イベント件数を集計 | 4枚のKPIカード |
| 直近イベントバナー | スナップショット内のイベントデータ | 直近7日のイベント企業を抽出しイベントスコア順に整形（Phase1は静的） | イベント企業バナー |
| スクリーニング実行／再読込 | 実行ボタン押下（対象＝母集団） | 母集団の構造×イベントを一括スコアリング→総合スコア再計算（非同期）。失敗・タイムアウト時はエラー化しリトライ可 | 成功：更新後のランキング・KPI／失敗：エラー表示＋再試行導線 |
| 企業ランキング表示 | スコアリング結果 | 既定で総合スコア降順にソート（構造/イベントへトグル可） | ランキングテーブル（確定列） |
| フィルタ・並び替え | 業種／時価総額帯／スコア帯／イベント有無、ソートキー | 条件で抽出・並び替え | 絞り込み後の一覧 |
| 企業検索（完全一致＋自然言語） | 企業名／証券コード、または自然言語クエリ（最大200文字、例「含み益500億円以上・関西・小売」） | 完全一致検索／自然言語はエージェントが条件抽出→母集団に適用。条件抽出できない場合は解釈失敗とする | 一致企業の一覧（自然言語時は絞り込み一覧＋要約）／解釈失敗：再入力を促す表示（結果0件とは区別） |
| 詳細へ遷移 | 企業行の選択 | 対象企業IDを保持して画面遷移 | P002（企業詳細） |
| ロングリストへ追加 | チェックした企業群 | 選択企業をロングリストに登録（重複排除） | P003へ追加・追加完了通知 |
| 確信度インジケータ | 企業の確信度／データ充足度 | しきい値未満を低確信度と判定 | スコアセルのインラインバッジ/トーン |
| イベントブースト倍率バッジ | 企業のイベント加点倍率 | 倍率を整形 | 企業名横のバッジ（例「×1.30」） |
| スコア変化の差分ハイライト（Phase2） | 今回／前回スコア（履歴） | 差分を算出し方向を判定 | 上昇/下降の矢印・色強調 |

## UI状態遷移

| 要素 | 対応機能 | 状態 |
|------|---------|------|
| KPIカード群 | KPIカード表示 | ローディング／表示／データなし |
| 新規イベントバナー | 直近イベントバナー | 表示／イベントなし |
| 実行・再読込ボタン | スクリーニング実行／再読込 | 有効／実行中（ローディング・disable）／完了／失敗（エラー表示＋再試行） |
| フィルタパネル | フィルタ・並び替え | 未適用／適用中／クリア |
| 並び替えトグル | フィルタ・並び替え | 総合スコア（既定）／構造スコア／イベントスコア |
| 検索ボックス | 企業検索 | 空／入力中（完全一致）／入力中（自然言語）／結果0件／解釈失敗（再入力促し）／文字数超過 |
| ランキングテーブル | 企業ランキング表示 | ローディング（スケルトン）／表示／0件／行hoverハイライト／行選択 |
| スコアセル | 確信度インジケータ | 通常／低確信度マーク |
| 企業名横バッジ | イベントブースト倍率バッジ | 倍率なし／倍率表示 |
| 差分インジケータ（Phase2） | スコア変化の差分ハイライト | 差分なし／上昇／下降 |
| チェックボックス＋追加ボタン | ロングリストへ追加 | 未選択（追加ボタン無効）／選択あり（有効）／追加中 |
