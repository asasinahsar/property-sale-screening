# 要件定義書: P000 ログイン（認証）

## 目的

許可されたユーザー（事業開発企画部の担当者・責任者）のみがシステムにアクセスできるよう認証を行う。あわせてロール（担当者／責任者）に応じた認可の基点とし、Sprint3 のセキュリティ評価軸（認証・認可）を実証する。

> **粒度の注記**: 本フェーズは概念レベルのIPO＋UI状態遷移。認証方式・セッション仕様・テーブル/APIの詳細は次のDB/API設計フェーズで追記する。

## 機能

| # | 機能 | 説明 | Phase | 仕様書からの展開／変更点 |
|---|------|------|-------|--------------|
| 1 | ログイン認証 | ユーザーID（またはメール）とパスワードで本人認証 | Phase1 | IPOと検証ルールを明文化 |
| 2 | ロールベース認可 | 認証成功時にロール（担当者／責任者）を判定し権限制御の基点とする | Phase1 | ロール保持先（セッション）を明記 |
| 3 | セッション管理 | セッション発行・有効期限・未認証/タイムアウト時のリダイレクト（復帰先URLを保持） | Phase1 | タイムアウト復帰挙動を追加 |
| 4 | 認証エラー表示 | 認証失敗・入力不備の通知（資格情報は晒さない） | Phase1 | エラー種別を区別 |
| 5 | ログアウト | セッション破棄してログイン画面へ | Phase1 | 変更なし |
| 6 | パスワードポリシー | 最低文字数・複数文字種・脆弱パスワード禁止を検証 | Phase1 | 新規（セキュリティ軸） |
| 7 | ログイン試行回数制限・アカウントロック | 連続失敗（例：5回）で一定時間アカウントを一時ロック | Phase1 | 新規（ブルートフォース対策） |
| 8 | 認証レート制限 | 認証リクエストのレート制限 | Phase1 | 新規（ブルートフォース対策） |

## Input / Process / Output

| 機能 | Input | Process | Output |
|------|-------|---------|--------|
| ログイン認証 | ユーザーID／メール、パスワード | 入力検証（必須・形式）→資格情報を照合（パスワードはハッシュ比較）→成否判定 | 成功：セッション発行しP001へ遷移／失敗：エラーメッセージ |
| ロールベース認可 | 認証済みユーザー情報 | ユーザーに紐づくロール（担当者／責任者）を取得しセッションに保持 | 以降画面の操作権限（例：P003承認可否）に反映 |
| セッション管理 | セッショントークン、アクセス先URL | トークン検証・有効期限チェック。タイムアウト/未認証時は復帰先URLを保持 | 有効：画面表示／無効・タイムアウト：P000へリダイレクト→再ログイン後に復帰先へ戻す |
| 認証エラー表示 | 認証結果（失敗理由） | 入力不備／認証失敗／試行超過/ロックを区別し汎用メッセージを生成 | エラーメッセージ表示（入力値は保持） |
| ログアウト | ログアウト操作 | セッション破棄 | P000へ遷移 |
| パスワードポリシー | パスワード | 最低文字数・文字種・脆弱パスワード辞書を検証 | 合致：認証続行／不合致：ポリシー違反メッセージ |
| ログイン試行回数制限・アカウントロック | 連続失敗回数 | しきい値（例5回）超過でアカウントを一定時間ロック | ロック中表示・解除時刻の案内 |
| 認証レート制限 | 認証リクエスト頻度 | 単位時間あたりのリクエスト数を制限 | 超過時は一時的に受付拒否 |

## UI状態遷移

| 要素 | 対応機能 | 状態 |
|------|---------|------|
| ユーザーID入力 | ログイン認証 | 通常／フォーカス／エラー（必須・形式不備） |
| パスワード入力 | ログイン認証 | 通常／フォーカス／マスク・表示切替／エラー |
| ログインボタン | ログイン認証 | 有効／無効（入力不足）／ローディング（認証中はdisable）／ロック中（操作不可） |
| パスワード入力 | パスワードポリシー | 通常／ポリシー違反エラー |
| エラーメッセージ | 認証エラー表示 | 非表示／表示（入力不備）／表示（認証失敗）／表示（試行回数超過）／表示（アカウントロック中・解除時刻） |
| セッション復帰 | セッション管理 | 通常／タイムアウト→再ログイン誘導→復帰先へ遷移 |
