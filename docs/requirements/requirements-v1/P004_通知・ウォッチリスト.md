# 要件定義書: P004 通知・ウォッチリスト（Phase2）

## 目的

担当者が関心企業をウォッチリストに登録し、スコアしきい値超え・新規開示・大量保有報告書の検知時にメール／Slack等へ通知する。継続モニタリング（Phase2）のUI受け皿。

> **スコア呼称**: 構造スコア（=定量）／イベントスコア（=定性）／総合スコア（統合した派生値・並び順キー）。
> **Phaseの注記**: 本画面の全機能は **Phase2（本番運用）**。Phase1（Sprint3 MVP）ではP001のスナップショット表示に留め、本画面は提供しない。
> **粒度の注記**: 本フェーズは概念レベルのIPO＋UI状態遷移。具体的なテーブル名・API仕様・通知連携の詳細は次のDB/API設計フェーズで追記する。検知元のスコア・イベントは [`scoring-and-pipeline.md`](./scoring-and-pipeline.md) を参照。

## 機能

| # | 機能 | 説明 | Phase | 備考 |
|---|------|------|-------|------|
| 1 | ウォッチリスト登録/解除 | 関心企業をウォッチリストに追加・除外 | Phase2 | P001/P002から登録動線 |
| 2 | 通知条件設定 | しきい値（例：総合/構造スコア）・イベント種別（新規開示・大量保有報告書等）を設定 | Phase2 | 企業単位/全体共通の条件 |
| 3 | 通知履歴一覧 | 過去の通知を時系列で一覧 | Phase2 | 既読/未読管理 |
| 4 | 通知からP002へ遷移 | 通知/履歴から該当企業の詳細へ | Phase2 | ディープリンク |

## Input / Process / Output

| 機能 | Input | Process | Output |
|------|-------|---------|--------|
| ウォッチリスト登録/解除 | 企業ID、登録/解除操作 | ウォッチ対象を更新 | 更新後のウォッチリスト |
| 通知条件設定 | しきい値・イベント種別 | 条件を保存し監視対象に反映 | 設定済み通知条件 |
| 通知トリガー（バックグラウンド） | スコア更新・新規開示・大量保有報告書検知（scoring-and-pipeline.md） | 条件に合致するか判定し通知を送出 | メール／Slack等への通知＋履歴記録 |
| 通知履歴一覧 | 履歴データ | 時系列で取得・整形 | 通知履歴リスト（既読/未読） |
| 通知からP002へ遷移 | 通知/履歴の選択 | 企業IDを保持して遷移 | P002 |

## UI状態遷移

| 要素 | 対応機能 | 状態 |
|------|---------|------|
| ウォッチ登録ボタン | ウォッチリスト登録/解除 | 未登録／登録済み／処理中 |
| 通知条件フォーム | 通知条件設定 | 未設定／編集中／保存中／保存済み |
| 通知履歴リスト | 通知履歴一覧 | ローディング／表示／0件／既読・未読 |
| 履歴アイテム | 通知からP002へ遷移 | 有効 |
