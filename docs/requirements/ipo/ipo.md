# Input / Process / Output 一覧

> 全機能のデータの流れと処理の内容を網羅する。スコア呼称＝構造スコア（定量）／イベントスコア（定性）／総合スコア（統合派生値）。
> 計算式の係数・テーブル・APIは次のDB/API設計フェーズで詳細化（本書は概念レベル）。

### 確認結果
- **重複の統合**：「ロングリストへ追加」は P001(C8)・P002(D10) の2入口だが処理・Outputは共通（E1へ集約）として整理。「詳細へ遷移」(C7/E7/F4) はナビゲーションとして各所に存在するが矛盾なし。
- **Input供給の充足**：各画面が必要とするスコア・指標・シグナル・確信度はすべて A（パイプライン）が供給元として存在し、欠落なし。A1の大量保有報告書は A4/A8 へ、C2のイベント記録は A3/A1 由来として供給経路を明示。
- **フィードバックループの明示**：A6（較正済みパラメータ）→A2/A4/A5、A7（更新済み重み・Phase2）→A4/A5 を追記し、検証・学習がスコアへ反映される経路を可視化。
- **自然言語検索の経路**：C6 は収集済みデータセット（A1企業マスタ＋A5スコア）への絞り込みであり、バッチ収集A1の再実行ではないことを明記。
- **実行権限の整合**：承認/却下＝責任者のみ（E3/E4）、スクリーニング実行(C3)・レポート出力(D9)・エクスポート(E6)＝担当者・責任者とも可、を粒度をそろえて明記。
- **矛盾なし**：Output→Input の連鎖（A→C/D/E、フィードバックA6/A7→A、認可B2→E3/E4、選定C8/D10→E1、検知A8→F）が一貫。
