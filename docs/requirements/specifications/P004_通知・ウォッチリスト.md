# 仕様書: P004 通知・ウォッチリスト（Phase2）

> **Phase**: 本画面の全機能は Phase2（本番運用）。Phase1（Sprint3 MVP）では提供せず、継続モニタリングは P001 のスナップショット表示に留める。

## 目的

担当者が関心企業をウォッチリストに登録し、スコアしきい値超え・新規開示・大量保有報告書の検知時にメール／Slack等へ通知する。継続モニタリング（Phase2）のUI受け皿。

## 機能一覧

| # | 機能 | 説明 |
|---|------|------|
| 1 | ウォッチリスト登録/解除 | 関心企業をウォッチリストに追加・除外（P001/P002から登録動線） |
| 2 | 通知条件設定 | しきい値（総合/構造スコア）・イベント種別（新規開示・大量保有報告書等）を設定 |
| 3 | 通知履歴一覧 | 過去の通知を時系列で一覧（既読/未読管理） |
| 4 | 通知からP002へ遷移 | 通知／履歴から該当企業の詳細へ |

## UI要素

| 要素 | 説明 | 位置 |
|------|------|------|
| ウォッチリストテーブル | 登録企業の一覧（企業名・スコア・通知条件・状態） | 画面中央 |
| ウォッチ登録/解除ボタン | 企業のウォッチ対象切替 | 各行／企業詳細 |
| 通知条件フォーム | しきい値・イベント種別の設定 | 上部または右サイド |
| 通知履歴リスト | 通知の時系列リスト（既読/未読） | 下部またはタブ |
| 履歴アイテム | 各通知（企業名・トリガー・日時） | 通知履歴リスト内 |

## 機能とUIの結びつき

| 機能 | UI要素 | インタラクション |
|------|--------|----------------|
| ウォッチリスト登録/解除 | ウォッチ登録/解除ボタン | 押下で対象を追加／除外 |
| 通知条件設定 | 通知条件フォーム | 入力・保存で監視条件を更新 |
| 通知履歴一覧 | 通知履歴リスト | 表示時にロード、既読/未読を切替 |
| 通知からP002へ遷移 | 履歴アイテム | クリックで P002 へ遷移 |
