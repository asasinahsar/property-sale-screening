# 仕様書: P000 ログイン（認証）

## 目的

許可されたユーザー（事業開発企画部の担当者・責任者）のみがシステムにアクセスできるよう認証を行う。あわせてロール（担当者／責任者）に応じた認可の基点とし、Sprint3 のセキュリティ評価軸（認証・認可）を実証する。

> 詳細なIPO・UI状態遷移は `requirements-v1/P000_ログイン.md` を参照。

## 機能一覧

| # | 機能 | 説明 | Phase |
|---|------|------|-------|
| 1 | ログイン認証 | ユーザーID（またはメール）とパスワードで本人認証する | Phase1 |
| 2 | ロールベース認可 | 認証成功時にロール（担当者／責任者）を判定し、以降の画面の操作権限を制御する基点とする | Phase1 |
| 3 | セッション管理 | セッション発行・有効期限・未認証/タイムアウト時のリダイレクト（復帰先URLを保持し再ログイン後に復帰） | Phase1 |
| 4 | 認証エラー表示 | 認証失敗・入力不備・試行超過/ロックの通知（資格情報は晒さない） | Phase1 |
| 5 | ログアウト | セッションを破棄してログイン画面へ戻す | Phase1 |
| 6 | パスワードポリシー | 最低文字数・複数文字種・脆弱パスワード禁止を検証 | Phase1 |
| 7 | ログイン試行回数制限・アカウントロック | 連続失敗（例5回）で一定時間アカウントを一時ロック | Phase1 |
| 8 | 認証レート制限 | 認証リクエストのレート制限（ブルートフォース対策） | Phase1 |

## UI要素

| 要素 | 説明 | 位置 |
|------|------|------|
| ユーザーID入力 | ユーザーID／メールアドレスの入力フィールド | 中央フォーム上部 |
| パスワード入力 | パスワード入力フィールド（マスク表示／表示トグル、ポリシー違反表示） | 中央フォーム中段 |
| ログインボタン | 認証を実行するボタン（ロック中は操作不可） | 中央フォーム下部 |
| エラーメッセージ | 認証失敗・入力不備・試行超過・ロック中（解除時刻）の通知領域 | フォーム上部 |
| アプリ名／ロゴ | システム名の表示 | フォーム上部ヘッダ |

## 機能とUIの結びつき

| 機能 | UI要素 | インタラクション |
|------|--------|----------------|
| ログイン認証 | ユーザーID入力／パスワード入力／ログインボタン | 入力後ボタン押下で認証実行。成功で P001 へ遷移 |
| ロールベース認可 | （内部処理） | 認証成功時にロールを保持し、P003 の承認操作可否などに反映 |
| 認証エラー表示 | エラーメッセージ | 認証失敗・試行超過・ロック中に表示、フォームは保持 |
| セッション管理 | （内部処理） | 未認証/タイムアウトで他画面アクセス時は本画面へリダイレクトし、再ログイン後に復帰先へ |
| パスワードポリシー | パスワード入力 | ポリシー違反時にエラー表示 |
| ログイン試行回数制限・アカウントロック | ログインボタン／エラーメッセージ | 連続失敗超過でロック中表示・ボタン操作不可 |
