# 仕様書（非画面）: スコアリング & データパイプライン

> **非画面コンポーネント**。各画面（P001〜P004）の Input となる「構造/イベント/総合スコア・指標・定性シグナル・確信度」を生成するバックエンド処理。UIを持たないため UI要素・遷移は定義しない。
> 詳細な処理要件は `requirements-v1/scoring-and-pipeline.md` を参照。具体的な計算式の係数・テーブル・APIは次のDB/API設計フェーズで詳細化する。

## 役割（機能一覧）

| # | 機能 | 説明 | Phase |
|---|------|------|-------|
| 1 | データ収集 | EDINET（有報・適時開示・大量保有報告書）、YFinance（株価・時価総額）から母集団データを取得 | Phase1 |
| 2 | 構造スコア算出 | 含み益倍率・修正PBR・自己資本比率・ROIC対WACC等を業種内z-scoreで正規化し合成（0–100） | Phase1 |
| 3 | 定性シグナル抽出（LLM） | 出力スキーマ固定（種別・強度・document_type・source_page・引用文必須）＋原文照合で出典の取れないシグナルを不採用 | Phase1 |
| 4 | イベントスコア算出 | シグナル種別×強度の重み付け、先行シグナル加点、反証減点、recency考慮 | Phase1 |
| 5 | 統合スコア・確信度算出 | 構造×イベントから総合スコア（並び順キー）、AI総合判定、確信度/データ充足度を生成 | Phase1 |
| 6 | 精度検証（バックテスト） | 過去売却企業でスコアの事前妥当性を検証し、しきい値・重みをキャリブレーション | Phase1 |
| 7 | ヒューマン・イン・ザ・ループ | 担当者の当たり/外れラベルで重みを再調整する学習ループ | Phase2 |
| 8 | 変化検知・通知トリガー | スコア更新・新規開示・大量保有報告書を検知し P004 の通知を駆動 | Phase2 |

## 各画面への供給データ

| 供給先 | 主な供給データ |
|--------|--------------|
| P001 | 構造/イベント/総合スコア、KPI集計、直近イベント、ブースト倍率、確信度 |
| P002 | スコア内訳、定量指標標準セット、ROIC/WACC、定性シグナル（支持/反証）、出典、確信度、AI総合判定 |
| P003 | 各スコア、含み益・倍率、出典サマリ |
| P004（Phase2） | スコア更新、新規開示・大量保有報告書の検知イベント |
